# Decision: turnStart 프롬프트에 지식 노트 탐색을 서브에이전트에 위임

## 결정

turnStart 프롬프트에 "지식 노트를 미리 읽고 요약하라"는 지시사항을 추가하되, 이 작업을 메인 에이전트가 직접 수행하지 않고 **explore 서브에이전트에 위임**하도록 명시한다.

## 근거

메인 에이전트가 직접 지식 노트를 읽는 것보다, **전문화된 탐색 서브에이전트(explore)**가 연관 노트를 파악하여 핵심만 요약해주는 것이 컨텍스트 관리와 파악에 훨씬 효율적이다.

- explore 에이전트는 "관련 노트 탐색 → 핵심 요약"에 특화된 워크플로우를 가짐
- 메인 에이전트는 불필요한 상세 내용을 읽느라 토큰을 낭비하지 않음
- 연결된 노트들의 맥락을 explore가 더 효과적으로 파악 가능

## 고려한 대안

- **메인 에이전트가 직접 읽기**: 탈락 — 토큰 낭비, 맥락 파악 비효율, 관련 노트 탐색 누락 가능성
- **turnStart에 요약된 인덱스만 주입**: 탈락 — 실제 노트 내용이 없으면 구체적인 맥락 활용 불가
